# HLTNC T3D — Полное руководство по управлению через Modbus RTU

**Привод:** HLTNC T3D-L20A-RABF  
**Мотор:** 80AST-A1C04025 (2000 об/мин, 2.4 Нм ном., 10000 имп/об энкодер)  
**Протокол:** Modbus RTU / RS-485

---

## 1. Физическое подключение

### 1.1 Разъёмы привода

Привод имеет два независимых интерфейса:

| Разъём | Тип | Назначение |
|--------|-----|------------|
| **RS-485 terminal** | 3-pin клеммник | Связь с ПК/контроллером (A, B, GND) |
| **CN1** | DB44 | DI/DO, аналог, энкодер, импульсный вход |

> **КРИТИЧНО:** RS-485 клеммник физически отдельный от CN1. Связь работает даже при полностью отключённом CN1.

### 1.2 Подключение USB-RS485 адаптера

```
PC (USB)
   │
[USB-RS485 адаптер]
   │
   ├── A  ──► RS-485 A (клеммник привода)
   ├── B  ──► RS-485 B (клеммник привода)
   └── GND ──► RS-485 GND (клеммник привода)
```

**Параметры адаптера:** не требуют настройки — протокол настраивается на стороне ПО.

**Параметры порта:**
- Baud rate: **19200**
- Data bits: **8**
- Parity: **Even (E)**
- Stop bits: **1**
- Slave ID: **1** (заводской дефолт)

### 1.3 Требования к CN1

**CN1 (DB44) должен быть отключён или не иметь ничего подключённого к пинам PULS+ и SIGN+.**

Причина: если на пинах 24 (SIGN+5V) или 25 (PULS+5V) есть любые провода (даже болтающиеся, не подключённые с другой стороны), привод при старте детектирует импульсный вход и принудительно выставляет `P-025=3` (режим импульсного входа), переопределяя любые настройки Modbus.

Допустимо иметь CN1 полностью отключённым — привод работает только через RS-485.

## 2. Первоначальная настройка (выполняется один раз)

Порядок важен — P-025 сохраняется в EEPROM только через альтернативный метод и только после того, как P-004=1 уже записан в EEPROM.

### Шаг 1 — Установить режим скорости и сохранить P-004

```
Записать P-004 = 1  (FC06, addr=4, val=1)
Сохранить EEPROM    (FC41)
Выключить питание   (подождать 5 секунд)
Включить питание
Проверить: FC03 addr=4 должен вернуть 1
```

### Шаг 2 — Установить источник скорости и сохранить P-025

```
Записать P-025 = 1  (FC06, addr=25, val=1)
Сохранить через АЛЬТЕРНАТИВНЫЙ метод: FC06 addr=4097 val=4660
                                       (0x1001 = 0x1234)
Выключить питание   (подождать 5 секунд)
Включить питание
Проверить: FC03 addr=25 должен вернуть 1
```

> **Почему два метода EEPROM?** Стандартный FC41 не всегда сохраняет P-025 корректно. Запись 0x1234 на адрес 0x1001 через FC06 — второй метод сохранения из раздела 6.6 документации Modbus V3.3.

### Шаг 3 — Принудительный servo-on

```
Записать P-098 = 1  (FC06, addr=98, val=1)
Сохранить EEPROM    (FC41)
```

После этого привод будет включать серво автоматически при подаче питания — физический сигнал SON на DI не нужен.

### Шаг 4 — Задать начальную скорость

```
Записать P-137 = 300  (FC06, addr=137, val=300)
Сохранить EEPROM      (FC41)
```

После этих шагов: при каждом включении питания мотор автоматически запустится на 300 об/мин.

### Итоговые параметры конфигурации

| Параметр | Адрес | Значение | Описание |
|----------|-------|----------|----------|
| P-004 | 4 | 1 | Режим управления: скорость |
| P-025 | 25 | 1 | Источник скорости: внутренние пресеты |
| P-046 | 46 | 300 | Коэффициент аналогового входа (не используется, но задан) |
| P-060 | 60 | 100 | Время разгона, мс/1000 об/мин |
| P-061 | 61 | 100 | Время торможения, мс/1000 об/мин |
| P-075 | 75 | 2600 | Максимальная скорость, об/мин |
| P-098 | 98 | 1 | Принудительный servo-on (без DI) |
| P-100 | 100 | 10 | Функция DI1 = SP1 (не используется, сигнал не подан) |
| P-137 | 137 | скорость | Пресет скорости 1 (активный при P-025=1 без SP-сигналов) |

## 3. Протокол Modbus

### 3.1 Функциональные коды

| FC | HEX | Операция | Примечание |
|----|-----|----------|------------|
| FC03 | 0x03 | Чтение параметров P-xxx | До 8 регистров за раз |
| FC04 | 0x04 | Чтение статусных регистров | До 8 за раз, read-only |
| FC06 | 0x06 | Запись одного параметра | Только в RAM; нужен FC41 для EEPROM |
| FC10 | 0x10 | Запись нескольких параметров | До 10 за раз |
| FC41 | 0x41 | Сохранение RAM → EEPROM | Ждать 5 секунд после |
| FC42 | 0x42 | Управление серво | 0x55=вкл, 0xAA=выкл |

### 3.2 Адресация параметров

**FC03 addr = номер параметра в десятичной системе.**

```
P-004 → addr = 4
P-025 → addr = 25
P-137 → addr = 137
```

Диапазон параметров FC03: `0x0000 .. 0x0249` (addr 0..585).  
FC03 `0x1000..0x1063` — зеркало статусных регистров (read-only, для SCADA-совместимости).

### 3.3 Запись параметра (FC06)

```
Запрос:  [SlaveID] 06 [AddrH] [AddrL] [ValH] [ValL] [CRC]
Ответ:   [SlaveID] 06 [AddrH] [AddrL] [ValH] [ValL] [CRC]  (эхо)
```

Пример: записать P-137=500 (addr=0x0089, val=0x01F4):
```
01 06 00 89 01 F4 xx xx
```

**Важно:** Запись в RAM вступает в силу немедленно. Запись во время работы серво возвращает exception 0x10 (параметр заблокирован). Перед изменением P-137 нужно остановить серво через FC42.

### 3.4 EEPROM сохранение (FC41)

```
Запрос:  [SlaveID] 41 00 01 xx xx
Ответ:   [SlaveID] 41 00 01 xx xx
Ждать:   минимум 5 секунд
```

### 3.5 Альтернативное EEPROM сохранение (для P-025)

```
FC06 запись: addr=0x1001 (4097 decimal), val=0x1234 (4660 decimal)
```

Используется когда FC41 не сохраняет конкретный параметр (особенно P-025).

### 3.6 Управление серво (FC42)

```
Включить:   [SlaveID] 42 08 55 xx xx
Выключить:  [SlaveID] 42 08 AA xx xx
```

При `P-098=1` серво включается автоматически при подаче питания. FC42 disable — временная остановка; после следующего FC42 enable мотор запустится на текущем P-137.

## 4. Карта статусных регистров FC04

Чтение: FC04, адреса `0x0000..0x002D` (0..45 decimal).

| Addr (dec) | Название | Единица | Примечание |
|-----------|---------|---------|------------|
| 0 | Скорость мотора | об/мин | int16, отрицательное = реверс |
| 1 | Входная импульсная команда L | имп | Только для P-004=0 |
| 2 | Входная импульсная команда H | имп | |
| 3 | Команда позиции L | имп | int16 |
| 4 | Команда позиции H | имп | int16 |
| 5 | Текущая позиция мотора L | имп | int16 |
| 6 | Текущая позиция мотора H | имп | int16 |
| 7 | Отклонение позиции L | имп | |
| 8 | Отклонение позиции H | имп | |
| 9 | Момент мотора | % от ном | int16, знак = направление |
| 10 | Пиковый момент за 1 сек | % | |
| 11 | Мгновенный ток | А×10 | 12 = 1.2 А |
| 12 | Пиковый ток за 1 сек | А×10 | |
| 13 | Частота импульсной команды | ×0.1 кГц | |
| 14 | Задание скорости | об/мин | int16 |
| 15 | Задание момента | % | int16 |
| 16 | Аналог. напряжение скорости | мВ | int16 |
| 17 | Аналог. напряжение момента | мВ | int16 |
| 18 | Состояние входов DI | биты | бит0=DI1..бит7=DI8 |
| 19 | Состояние выходов DO | биты | бит0=DO1..бит5=DO6 |
| 20 | Позиция энкодера (1 оборот) L | имп | |
| 21 | Позиция энкодера (1 оборот) H | имп | |
| 22 | Многооборотная позиция | — | 0 если нет multi-loop энкодера |
| 23 | Загрузка тормозного резистора | % | |
| 24 | Средняя нагрузка | % | |
| 25 | Выходное напряжение | % | |
| 26 | Код аварии | — | **0 = норма** |
| 27 | Скорость мотора (точная) | ×0.1 об/мин | int16 |
| 40 | Напряжение шины DC | В | норма ~295–310 В |

**32-битная позиция (объединить addr 31 и 32):**
```
position_int32 = int32(FC04[32]) << 16 | FC04[31]
```

### 4.1 DO состояние (addr=19, биты)

| Бит | DO | Типичная функция |
|-----|-----|-----------------|
| 0 | DO1 | RDY (привод готов) |
| 1 | DO2 | — |
| 2 | DO3 | — |
| 3 | DO4 | BRK (тормоз разомкнут = 1) |
| 4 | DO5 | — |
| 5 | DO6 | — |

Нормальный рабочий DO: `0b001001` = DO1(RDY)=1, DO4(BRK)=1.

## 5. Ключевые параметры для управления

### 5.1 Скоростные пресеты (P-137..P-144)

При `P-025=1` и отсутствии физических SP-сигналов на DI, привод использует **P-137** как активную скорость.

| Параметр | Addr | Описание |
|----------|------|----------|
| P-137 | 137 | Скорость пресета 1 (активна по умолчанию) |
| P-138 | 138 | Скорость пресета 2 |
| P-139 | 139 | Скорость пресета 3 |
| P-140..P-144 | 140..144 | Пресеты 4..8 |

**Знак значения определяет направление:** положительное = вперёд, отрицательное = реверс.  
Диапазон: `-P-075` .. `+P-075` (максимальная скорость).

**Запись P-137 разрешена только при остановленном серво (FC42 disable).**

### 5.2 Управление разгоном/торможением

| Параметр | Addr | Ед. | Default | Описание |
|----------|------|-----|---------|----------|
| P-060 | 60 | мс/1000 об | 100 | Время разгона |
| P-061 | 61 | мс/1000 об | 100 | Время торможения |

Пример: разгон от 0 до 1000 об/мин за 200 мс → P-060 = 200.

### 5.3 Параметры момента/тока

| Параметр | Addr | Описание |
|----------|------|----------|
| P-069 | 69 | Ограничение момента (% от ном.), default=300 |
| P-075 | 75 | Максимальная скорость, об/мин |

## 6. Типичные операции (примеры команд)

### Запустить мотор на 500 об/мин

```
1. FC42 disable   → остановить (0xAA)
2. FC06 addr=137 val=500  → установить P-137=500
3. FC42 enable    → запустить (0x55)
4. FC04 addr=0    → проверить скорость
```

### Остановить мотор

```
FC42 disable  (0xAA)
```

### Реверс на 300 об/мин

```
1. FC42 disable
2. FC06 addr=137 val=65236   (= uint16(-300), реверс 300 об/мин)
3. FC42 enable
```

### Прочитать позицию (32-bit)

```
FC04 addr=31 → LowWord
FC04 addr=32 → HighWord
position = int32(HighWord<<16 | LowWord)  // в энкодерных импульсах
```

Импульсов на оборот: **10000** (энкодер 2500 линий × 4 = 10000 имп/об для 80AST).

### Сохранить все настройки

```
FC41  → сохранить
Ждать 5 секунд
Выключить/включить питание для проверки
```

## 7. Коды аварий (FC04 addr=26)

| Код | Описание | Действие |
|-----|----------|----------|
| 0 | Норма | — |
| 1 | Overcurrent | Проверить нагрузку, момент |
| 2 | Overvoltage | Проверить питание DC (норма 295–310 В) |
| 3 | Undervoltage | То же |
| 5 | Overload | Снизить нагрузку |
| 11 | Encoder error | Проверить кабель энкодера |
| 40/41 | Parameter error | Factory reset через панель |

Сброс аварии: FC06 addr=101 val=2 (DI2=ACLR) или через панель.

## 8. Интерфейс высокоуровневого драйвера

Этот раздел описывает API для библиотеки, которая инкапсулирует прямые Modbus-команды.

### 8.1 Типы данных

```go
type Status struct {
    SpeedRPM        float64  // текущая скорость, об/мин (знак = направление)
    SpeedSetpoint   int      // задание скорости, об/мин
    PositionPulses  int64    // абсолютная позиция, энкодерные импульсы
    TorquePercent   int      // момент, % от номинального (знак = направление)
    CurrentAmps     float64  // ток, А
    DCBusVoltage    int      // напряжение шины DC, В
    AlarmCode       int      // код аварии (0 = норма)
    ServoReady      bool     // DO1 = RDY
    BrakeReleased   bool     // DO4 = BRK released
}

type MoveParams struct {
    SpeedRPM        int     // скорость вращения (всегда положительная)
    Direction       int     // +1 = вперёд, -1 = реверс
    AccelMs         int     // время разгона мс/1000 об (0 = не менять)
    DecelMs         int     // время торможения мс/1000 об (0 = не менять)
}
```

### 8.2 Основные методы

```go
type T3DDriver interface {
    // --- Жизненный цикл ---
    
    Connect(port string) error
    Close() error
    
    // --- Статус ---
    
    // ReadStatus читает все FC04 регистры за один запрос
    ReadStatus() (Status, error)
    
    // GetAlarm возвращает код аварии (0 = норма)
    GetAlarm() (int, error)
    
    // --- Базовое управление ---
    
    // SetSpeed устанавливает P-137 (останавливает серво на время записи)
    SetSpeed(rpm int) error
    
    // Enable включает серво (FC42 0x55)
    Enable() error
    
    // Disable выключает серво (FC42 0xAA), мотор тормозит по P-061
    Disable() error
    
    // --- Позиционирование ---
    
    // RotateByPulses вращает на deltaPulses энкодерных импульсов с заданной скоростью.
    // Блокирует до завершения или ошибки. Точность: ±(speedRPM/60 * 10000 * pollInterval).
    RotateByPulses(pulses int64, params MoveParams) error
    
    // RotateByTurns вращает на turns оборотов (1 оборот = 10000 имп)
    RotateByTurns(turns float64, params MoveParams) error
    
    // GetPosition возвращает абсолютную позицию в энкодерных импульсах
    GetPosition() (int64, error)
    
    // ResetPosition сбрасывает счётчик позиции (если поддерживается прошивкой)
    ResetPosition() error
    
    // --- Момент / натяжение ---
    
    // GetTorque возвращает текущий момент в % от номинального
    // Знак соответствует направлению силы
    GetTorque() (int, error)
    
    // GetTorqueNewtonMeters возвращает момент в Нм
    // Для 80AST-A1C04025 номинальный момент = 2.4 Нм
    GetTorqueNewtonMeters() (float64, error)
    
    // HoldTension удерживает заданное натяжение (момент) в % от номинала.
    // Скорость ограничена maxSpeedRPM. Работает в режиме регулятора:
    // если момент < target → увеличивает скорость, если > target → уменьшает.
    // Выход: по контексту (ctx.Done) или по аварии.
    HoldTension(ctx context.Context, targetTorquePercent int, maxSpeedRPM int) error
}
```

### 8.3 Реализация RotateByPulses

```
1. Прочитать стартовую позицию: GetPosition() → startPos
2. Определить targetPos = startPos + pulses (с учётом направления)
3. Установить скорость: SetSpeed(params.SpeedRPM * params.Direction)
4. Enable()
5. Цикл опроса (каждые ~20 мс):
   a. ReadStatus() → currentPos, alarmCode
   b. if alarmCode != 0 → Disable(), вернуть ошибку
   c. if |currentPos - startPos| >= |pulses| → Disable(), выйти
   d. if |targetPos - currentPos| < DECEL_THRESHOLD → плавно снизить SetSpeed
6. Вернуть итоговую позицию
```

**Точность позиционирования** при 300 об/мин и интервале опроса 20 мс:
- 300 об/мин = 5 об/с = 50000 имп/с
- За 20 мс проходит ~1000 имп (0.1 оборота)
- Для высокой точности нужно снижать скорость заблаговременно

### 8.4 Натяжение через момент

Привод в режиме скорости измеряет момент в реальном времени (FC04 addr=9). При намотке кабеля или ленты:

```
F (сила натяжения, Н) = T (Нм) / r (радиус барабана, м)

T = (TorquePercent / 100) × T_nominal

T_nominal (80AST-A1C04025) ≈ 2.4 Нм
```

Пример реализации HoldTension:
```
целевой момент = 50% → целевая сила = (0.5 × 2.4) / r
цикл:
    torque = GetTorque()
    if torque < target - deadband:
        SetSpeed(currentSpeed + delta)
    elif torque > target + deadband:
        SetSpeed(currentSpeed - delta)
    if currentSpeed <= 0:
        Disable()  // нет слабины — остановить
    sleep(controlInterval)
```

## 9. Диагностика и частые проблемы

### P-025 откатывается к 3 после перезапуска

**Причина:** На CN1 пины PULS+/SIGN+ (24/25) подключено что-либо — даже провода без сигнала.  
**Решение:** Полностью отключить CN1 или убедиться что пины 24/25 не имеют никаких проводов.  
**Метод сохранения:** Использовать альтернативный EEPROM save (FC06 addr=4097 val=4660), а не FC41.

### Привод не реагирует на запись P-137 (exception 0x10)

**Причина:** Параметр заблокирован во время работы серво.  
**Решение:** FC42 disable → записать P-137 → FC42 enable.

### Скорость 0 несмотря на P-137 > 0

Проверить по порядку:
1. FC03 addr=25 = 1? (P-025 = 1, не 3)
2. FC03 addr=4 = 1? (P-004 = 1, не 0 или 3)
3. FC03 addr=98 = 1? (P-098 = 1)
4. FC04 addr=26 = 0? (нет аварии)
5. CN1 PULS+/SIGN+ чистые? (см. выше)

### CRC ошибки / таймауты

**Причина 1:** Неправильная полярность RS-485 (A и B перепутаны).  
**Причина 2:** Нет общего GND между адаптером и приводом.  
**Причина 3:** Слишком длинный кабель без терминирующего резистора 120 Ом.

### Мотор крутится не в ту сторону

P-137 = отрицательное значение для реверса:
```
FC06 addr=137 val=65236   // = uint16(-300) = реверс 300 об/мин
```
Или установить P-048=1 (инверсия направления) в EEPROM.

# EEPROM сохранение (стандартный)
go run cmd/main.go -slave 1 -port /dev/ttyUSB0 -save-eeprom

# EEPROM сохранение (альтернативный, для P-025)
go run cmd/main.go -slave 1 -port /dev/ttyUSB0 -write-param -addr 4097 -val 4660

# Серво вкл/выкл
go run cmd/main.go -slave 1 -port /dev/ttyUSB0 -servo-enable
go run cmd/main.go -slave 1 -port /dev/ttyUSB0 -servo-disable

# Чтение диапазона параметров
go run cmd/main.go -slave 1 -port /dev/ttyUSB0 -scan-raw -from-addr 4 -to-addr 4

# Чтение статусных регистров
go run cmd/main.go -slave 1 -port /dev/ttyUSB0 -scan-status -from-addr 0 -to-addr 27
