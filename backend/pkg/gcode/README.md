# pkg/gcode — G-code Parser

Parses a subset of RS-274/NGC G-code into a flat, ordered list of motion commands.
Has **no dependencies** on hardware, robot, or motion packages — it is pure string
processing. Can be imported and tested on any platform without connected hardware.

Supported codes: `G0` (rapid), `G1` (linear feed), `G28` (home). All other G/M/S/T
codes are silently ignored (forward-compatible). Modal G0/G1 state and feed rate `F`
persist across lines.

---

## Import

```go
import "github.com/losion445-max/motor-control-hub-v2/pkg/gcode"
```

## Quick start

```go
package main

import (
    "fmt"
    "math"
    "log"
    "github.com/losion445-max/motor-control-hub-v2/pkg/gcode"
)

func main() {
    prog := `
; Home, then trace a rectangle at 300 mm/min
G28
G0 X100 Y100
G1 F300
X500 Y100
X500 Y400
X100 Y400
X100 Y100
G0 X0 Y0
`
    cmds, err := gcode.Parse(prog)
    if err != nil {
        log.Fatal(err)
    }
    for _, c := range cmds {
        x, y := "–", "–"
        if !math.IsNaN(c.X) { x = fmt.Sprintf("%.1f", c.X) }
        if !math.IsNaN(c.Y) { y = fmt.Sprintf("%.1f", c.Y) }
        fmt.Printf("G%d  X=%s  Y=%s  F=%.0f\n", c.Motion, x, y, c.F)
    }
}
```

Output:

```
G28  X=–  Y=–  F=0
G0   X=100.0  Y=100.0  F=0
G1   X=–  Y=–  F=300
G1   X=500.0  Y=100.0  F=0
G1   X=500.0  Y=400.0  F=0
G1   X=100.0  Y=400.0  F=0
G1   X=100.0  Y=100.0  F=0
G0   X=0.0  Y=0.0  F=0
```

## API reference

### `Motion` type and constants

```go
type Motion int

const (
    Rapid  Motion = 0   // G0 — rapid positioning
    Linear Motion = 1   // G1 — feed-rate linear move
    Home   Motion = 28  // G28 — return to home
)
```

### `Cmd`

A single resolved motion command. Produced by `Parse`.

```go
type Cmd struct {
    Motion Motion   // what to do
    X, Y   float64  // target position in mm; math.NaN() if not specified
    F      float64  // feed rate in mm/min; 0 = no update on this line
}
```

`HasX()` and `HasY()` are helpers that return `!math.IsNaN(c.X/Y)`. Use them
instead of comparing directly with `math.NaN()`.

### `Parse(src string) ([]Cmd, error)`

Parses a complete G-code program. Lines are processed top-to-bottom. A `Cmd` is
emitted when any of the following are true:

- The line specifies `G28` (home — no position needed).
- The line specifies `X` or `Y` (a position to move to).
- The line sets `F > 0` (a feed rate update with no position — runner must see it).

Blank lines and comment-only lines are skipped. Unknown G codes are silently ignored.

Returns an error only for malformed number literals (e.g. `X1.2.3`).

## Syntax supported

| Feature | Example | Notes |
|---|---|---|
| Spaced words | `G1 X100 Y200 F300` | Standard format |
| Compact words | `G1X100Y200F300` | No spaces needed |
| Line numbers | `N10 G0 X50` | N word ignored |
| Semicolon comment | `G0 X100 ; comment` | Rest of line discarded |
| Block comment | `G0 X100 (skip) Y200` | Parenthesised block removed |
| Modal G0/G1 | `G1\nX100 Y200` | G1 persists to next line |
| Signed coords | `G0 X-100 Y+200` | Leading sign accepted |
| Decimal coords | `G1 X100.5 Y200.75` | Float precision |
| Mixed case | `g1 x100 y200` | Normalised to upper |

## Implementation notes

**Modal state** starts in `G0` (Rapid). A `G1` or `G0` word updates the modal state
for all subsequent lines until changed again. `G28` also sets modal state to Home
but is treated as a one-shot command by `runner`.

**F-only lines** (`G1 F600` with no X/Y) are emitted as a `Cmd` with `HasX()` and
`HasY()` both false and `F = 600`. The runner picks up the feed rate from this
command even though it produces no physical movement. This matches standard G-code
behaviour where `F` is a modal parameter.

**Unsupported codes** (G90, G91, M3, S, T, etc.) are silently skipped. This
makes programs generated by CAM tools mostly compatible — motion-relevant lines
pass through while spindle/tool-change/mode lines are ignored.

**2-axis only.** This parser only handles `X` and `Y`. Z, A, B, C words are
ignored. The CDPR operates in a 2D plane so this is intentional.
