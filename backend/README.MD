# HLTNC T3D — Modbus RTU RS-485 Scanner (Go)

Скрипт для опроса драйвера серводвигателя **HLTNC T3D-L20A-RABF**
(и мотора **80AST-A1C04025**) через USB→RS-485 адаптер.

## Стек
- Порт: `/dev/ttyUSB0`
- Протокол: **Modbus RTU**
- Библиотека: `github.com/goburrow/serial` (низкоуровневый serial)
- Параметры по умолчанию T3D: **19200 baud, 8E1** (P-182=2, P-183=1)

---

## ⚠️ ВАЖНО — сначала настрой драйвер

По умолчанию коммуникация на T3D **выключена** (P-181 = -1).

Через панель на драйвере выставь:

| Параметр | Значение | Смысл |
|----------|----------|-------|
| P-181 | 1 | Slave ID = 1 (можно 1..32) |
| P-182 | 2 | Baud 19200 bps |
| P-183 | 1 | 8 бит, Even parity, 1 stop |

После изменения — сохрани параметры и перезапусти драйвер.

## Установка и запуск

```bash
# 1. Установи Go (если нет)
sudo apt install golang-go

# 2. Добавь себя в группу dialout (права на /dev/ttyUSB0)
sudo usermod -aG dialout $USER
newgrp dialout        # или перелогинься

# 3. Войди в папку
cd modbus-scanner

# 4. Скачай зависимости
go mod tidy

# 5. Запуск
go run main.go                              # сканировать все slave 1..32
go run main.go -slave 1                     # опросить конкретный slave
go run main.go -slave 1 -read-status        # только статус (скорость, ток, позиция)
go run main.go -slave 1 -read-params        # только параметры
go run main.go -v -slave 1                  # + сырые байты (hex)

# Другой порт / baud:
go run main.go -port /dev/ttyUSB1 -baud 9600 -slave 1

# Записать параметр (например сменить baud на 9600):
go run main.go -slave 1 -write-param -addr 182 -val 1

# Собрать бинарник:
go build -o t3d-scanner .
./t3d-scanner -slave 1
```

## Схема подключения RS-485

```
USB-адаптер              T3D-L20A (CN3)
    A  ───────────────────  A  (pin 1)
    B  ───────────────────  B  (pin 2)
   GND ───────────────────  GND(pin 3)
```

> Если кабель > 5 м — поставь терминирующий резистор **120 Ом** между A и B
> на дальнем конце линии.

## Карта регистров T3D

### FC04 — статусные значения (Input Registers, только чтение)

| Адрес | Имя    | Описание |
|-------|--------|----------|
| 0     | ST-000 | Скорость мотора (об/мин) |
| 1-2   | ST-001/002 | Текущая позиция (32 бит) |
| 3-4   | ST-003/004 | Задание позиции (32 бит) |
| 5     | ST-005 | Ошибка позиции (импульсы) |
| 6     | ST-006 | Ток фазы A (А×10) |
| 7     | ST-007 | Ток фазы B (А×10) |
| 8     | ST-008 | Момент (% от ном.) |
| 9     | ST-009 | Напряжение шины DC (В×10) |
| 10    | ST-010 | Температура радиатора (°C) |
| 11    | ST-011 | Статус I/O (битовая маска) |
| 12    | ST-012 | Код текущей ошибки (0 = норм) |
| 14    | ST-014 | Статус драйвера (битовая маска) |

### FC03 — параметры (Holding Registers, чтение/запись)

Адрес регистра = десятичный номер параметра P-xxx.

| Адрес | Параметр | Описание |
|-------|----------|----------|
| 120   | P-120    | Режим управления (3 = Modbus) |
| 130   | P-130    | Макс. скорость |
| 140   | P-140    | Направление вращения |
| 162   | P-162    | Скорость позиционирования |
| 181   | P-181    | Slave ID |
| 182   | P-182    | Baud rate |

Полный список — в debugging manual T3D V2.3.

## Функциональные коды Modbus (T3D)

| FC  | Назначение |
|-----|-----------|
| 03H | Читать параметр P-xxx |
| 04H | Читать статусное значение |
| 06H | Записать один параметр |
| 10H | Записать несколько параметров |
| 41H | Сохранить параметры в EEPROM |

## Частые проблемы

| Симптом | Причина | Решение |
|---------|---------|---------|
| "нет ответа" везде | P-181 = -1 | Выстави P-181 = 1 через панель |
| "нет ответа" у части | Неверный baud | Проверь P-182 = соответствует -baud |
| CRC mismatch | Parity | P-183=1 → 8E1, P-183=0 → 8N1 |
| Permission denied | Нет прав | `sudo usermod -aG dialout $USER` |
| Устройство не видно | Нет драйвера | `lsusb`, `dmesg | grep tty` |
