# Инструкция по монтажу и запуску кабельного робота

---

## Часть 1 — Механика

### 1.1 Рама

Сварить или собрать из профиля прямоугольную раму **1400 × 2400 мм**. Материал — стальной профиль 40×40 или 60×60 мм, достаточно жёсткий чтобы не прогибался под натяжением кабелей. Рама должна стоять вертикально или лежать горизонтально — в зависимости от задачи. Все четыре угла должны быть в одной плоскости, отклонение не более 5 мм.

### 1.2 Крепление моторов

Четыре мотора крепятся строго в углах рамы — ось барабана параллельна плоскости рамы:

- **M1** — верхний левый угол (0, 0)
- **M2** — верхний правый угол (1400, 0)
- **M3** — нижний правый угол (1400, 2400)
- **M4** — нижний левый угол (0, 2400)

Барабан должен быть направлен так, чтобы кабель уходил в сторону рабочей зоны, без перегибов. Крепить на сварных кронштейнах или болтами М8 через фланец мотора. Момент затяжки по документации на мотор.

### 1.3 Намотка кабелей

Использовать стальной трос или кевларовый шнур диаметром 3–5 мм. Намотка **однослойная** — это принципиально для постоянства передаточного отношения.

Порядок намотки:

1. Намотать кабель на барабан с небольшим преднатягом вручную — достаточно чтобы не соскользнул
2. Провести кабель через направляющий ролик к точке крепления на каретке камеры
3. Зафиксировать на каретке зажимным наконечником или узлом через коуш
4. Повторить для всех четырёх моторов

Длина кабеля при намотке — с запасом: от нуля до максимальной длины диагонали рамы плюс 20%. Для рамы 1400×2400: диагональ ≈ 2770 мм, запас → минимум 3.5 метра кабеля на каждом барабане.

### 1.4 Каретка камеры

Лёгкая платформа (алюминий или углепластик) с четырьмя точками крепления кабелей — по углам каретки. Точки крепления должны быть разнесены как можно дальше друг от друга для устойчивости. Установить камеру по центру каретки. Общая масса каретки с камерой не должна превышать расчётную нагрузку (для данных моторов — до 5 кг с разумным запасом).

## Часть 2 — Электрика

### 2.1 Силовое питание приводов

Каждый привод T3D питается от **однофазной сети 220 В** или от DC-шины. Подключить четыре привода к питанию согласно схеме в документации на T3D. Обязательно заземление корпусов приводов и рамы. Автоматы защиты — по одному на каждый привод, 6–10 А.

### 2.2 Подключение моторов к приводам

Каждый мотор 80AST соединяется со своим приводом T3D силовым кабелем (U/V/W фазы) и кабелем энкодера. Разъёмы фирменные — перепутать фазы можно, это изменит направление вращения. Если после первого запуска мотор вращается в обратную сторону — поменять местами любые две из трёх фаз (U-V, V-W или U-W).

### 2.3 RS-485 шина

Все четыре привода соединяются **последовательно** двумя проводами A и B:

```
Компьютер
  └─ USB → RS-485 адаптер (CH340 / FT232 / FTDI)
               ├─ A, B → Привод 1 (M1, верх-лево)
               ├─ A, B → Привод 2 (M2, верх-право)
               ├─ A, B → Привод 3 (M3, низ-право)
               └─ A, B → Привод 4 (M4, низ-лево)
                              └─ 120 Ом между A и B (терминирующий резистор)
```

Использовать витую пару. Экран подключить к земле со стороны компьютера. Длина шины — до 100 м при 19200 бод без проблем.

Терминирующий резистор 120 Ом ставится только на **последнем** приводе в цепочке.

## Часть 3 — Настройка приводов

Это делается **один раз** через панель управления на каждом приводе или через Modbus. Для каждого привода по очереди:

| Параметр | Значение | Что делает |
|---|---|---|
| P-181 | 1 / 2 / 3 / 4 | Адрес на шине Modbus (по номеру мотора) |
| P-182 | 2 | Скорость 19200 бод |
| P-183 | 1 | Формат 8E1 |
| P-004 | 1 | Режим управления скоростью |
| P-025 | 1 | Источник скорости — внутренний пресет |
| P-098 | 1 | Servo-on всегда включён |

После установки параметров — сохранить в EEPROM (на панели или через программу). Параметры сохраняются и не сбрасываются при отключении питания.

Проверить направление вращения: подать небольшую положительную скорость (P-137 = 10), включить серво. Барабан должен **наматывать** кабель (тянуть его к себе). Если разматывает — поменять две фазы мотора.

## Часть 4 — Программная часть

### 4.1 Установка

```bash
git clone <репозиторий>
cd motor-control-hub-v2/backend
go build ./...
```

Go 1.21 или новее. Зависимостей кроме `github.com/goburrow/modbus` нет — подтянется автоматически.

### 4.2 Проверить порт

```bash
ls /dev/ttyUSB*   # должен появиться после подключения адаптера
```

Если нет — проверить драйвер CH340: `sudo modprobe ch341`. Права на порт: `sudo usermod -aG dialout $USER`, перелогиниться.

### 4.3 Конфиг под реальную установку

```go
cfg := robot.Config{
    WidthMM:         1400,
    HeightMM:        2400,
    DrumRadiusMM:    67.8,    // замерить реальный радиус барабана
    HomingRPM:       15,      // медленно при хоминге
    HomingTorquePct: 18,      // порог натяжения при хоминге
    AccelMmPerSec2:  500,
    TorqueSafetyPct: 75,
    HoldTensionPct:  25,
    HoldTensionRPM:  10,
}
```

`DrumRadiusMM` — замерить физически штангенциркулем по внешней поверхности намотанного одного слоя кабеля.

## Часть 5 — Первый запуск

### Шаг 1 — Позиционирование камеры

Вручную, не включая приводы, перевести камеру примерно в **центр рабочей зоны** (700, 1200 мм от верхнего левого угла). Точность — в пределах 100–150 мм. Кабели должны быть слегка натянуты — не провисать, но и не тянуть с усилием.

### Шаг 2 — Включить питание приводов

Все четыре привода включить одновременно. Подождать 2–3 секунды до инициализации (индикаторы на приводах перестают мигать).

### Шаг 3 — Запустить хоминг

```go
sys := robot.NewSystem("/dev/ttyUSB0", 19200, cfg)
sys.Connect()
ctx := context.Background()
sys.Home(ctx)
```

Во время хоминга все четыре мотора медленно наматывают кабель. Каждый останавливается, когда натяжение достигает порога. Занимает 10–30 секунд. После окончания система считает текущую позицию равной (700, 1200).

**Если мотор не останавливается** — кабель провисает или порог момента слишком высок. Остановить вручную (Ctrl+C), проверить механику.

### Шаг 4 — Тестовое движение

```go
sys.MoveTo(ctx, 750, 1250, 20)  // 50 мм от центра, медленно
```

Камера должна плавно сместиться. Если дёргается или едет не в ту сторону — проверить направление вращения моторов.

### Шаг 5 — Полная программа

```go
prog := `
G28
G0 X140 Y240
G1 F1200
X1260 Y240
Y480
X140 Y480
...
`
cmds, _ := gcode.Parse(prog)
runner.Run(ctx, sys, cmds, runner.DefaultOpts)
```

## Что проверить если что-то пошло не так

| Симптом | Причина | Решение |
|---|---|---|
| Привод не отвечает на Modbus | Неверный P-181 или адрес шины | Проверить P-181 на приводе |
| Мотор крутится в обратную сторону | Перепутаны фазы | Поменять U и V местами |
| Хоминг не заканчивается | Кабель не натягивается | Уменьшить HomingRPM, проверить механику |
| Аварийный стоп во время движения | Перегрузка по моменту | Уменьшить TorqueSafetyPct или скорость |
| Камера едет не по прямой | Неверный DrumRadiusMM | Перемерить радиус барабана |
| После хоминга позиции неправильные | Камера была не в центре | Повторить хоминг с правильным позиционированием |
