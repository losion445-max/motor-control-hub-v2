# HLTNC T3D — Карта регистров и режимы управления

Драйвер: **HLTNC T3D-L20A-RABF**  
Мотор: **80AST-A1C04025**  
Связь: Modbus RTU, `/dev/ttyUSB0`, 19200 baud, 8E1, Slave ID=1  
Документ: **Modbus Communication function Description V3.3** (файл в корне проекта)

---

## Правило адресации

**FC03 (параметры) addr = номер параметра P-xxx в decimal.**  
Источник: V3.3, раздел 4 — "mailing address of parameter P-005 is 005 (decimal)"

| P-xxx | FC03 addr | Пример |
|-------|-----------|--------|
| P-004 | 4 | P-004 → addr=4 |
| P-181 | 181 | P-181 → addr=181 |

---

## FC03 — Ключевые параметры для управления (Holding Registers, R/W)

| P-xxx | FC03 addr | Описание | Значения |
|-------|-----------|----------|----------|
| P-004 | 4 | Режим управления | 0=позиция (импульсы), 1=скорость, 2=момент |
| P-005 | 5 | Kp контура скорости | Гц, default=50 |
| P-006 | 6 | Ti контура скорости | мс, default=20 |
| P-009 | 9 | Kp контура положения | 1/с, default=40 |
| P-025 | 25 | Источник команды скорости | 0=аналог, 1=внутренняя мультискорость, 3=импульс |
| P-060 | 60 | Время разгона | мс/1000об, default=100 |
| P-061 | 61 | Время торможения | мс/1000об, default=100 |
| P-075 | 75 | Максимальная скорость | об/мин, default=3000 |
| P-076 | 76 | Скорость JOG | об/мин, default=100 |
| P-098 | 98 | Принудительный servo-on | 0=внешний SON, **1=всегда ON** |
| P-100 | 100 | Функция DI1 | 1=SON, 10=SP1, 11=SP2, 12=SP3... |
| P-137 | 137 | Внутренняя скорость 1 | об/мин (для P-025=1) |
| P-138 | 138 | Внутренняя скорость 2 | об/мин |
| P-172 | 172 | Линии энкодера | default=2500 → 10000 имп/об |
| P-181 | 181 | Slave ID | 1..32 |
| P-182 | 182 | Baud rate | 0..5 (2=19200) |
| P-183 | 183 | Data format | 0..5 (1=8E1) |

## Функциональные коды

| FC | HEX | Описание |
|----|-----|----------|
| FC03 | 0x03 | Читать параметры P-xxx (до 8 за раз) |
| FC04 | 0x04 | Читать статус (до 8 за раз) |
| FC06 | 0x06 | Записать один параметр |
| FC10 | 0x10 | Записать несколько параметров (до 10) |
| FC41 | 0x41 | Сохранить параметры в EEPROM (ждать 5с) |
| FC42 | 0x42 | Servo enable (0x55) / disable (0xAA) |

**Альтернативный способ сохранить в EEPROM:** FC06, addr=0x1001, value=0x1234

## Режим 1 — Управление через импульсы (P-004 = 0)

Мотор получает команды через аппаратные входы Step/Dir (разъём CN1).  
Modbus — только чтение статуса и изменение параметров.

```bash
go run cmd/main.go -slave 1 -write-param -addr 4 -val 0
# После записи — выключить и включить питание
```

---

## Режим 2 — Управление скоростью через внутренние пресеты (P-004=1, P-025=1)

Управление скоростью через заданные Modbus значения P-137..P-144.  
Требует аппаратного DI (SP1/SP2/SP3) для выбора пресета.

```bash
# 1. Установить режим скорости
go run cmd/main.go -slave 1 -write-param -addr 4 -val 1
# 2. Источник — внутренняя мультискорость
go run cmd/main.go -slave 1 -write-param -addr 25 -val 1
# 3. Принудительный servo-on (без аппаратного SON)
go run cmd/main.go -slave 1 -write-param -addr 98 -val 1
# 4. Задать скорость пресета 1
go run cmd/main.go -slave 1 -write-param -addr 137 -val 300
# 5. Сохранить в EEPROM
go run cmd/main.go -slave 1 -save-eeprom
```

> DI1 должен быть на +24В (функция SP1=10) для выбора пресета 1.

## Важные ограничения

1. **Прямой позиционной команды через Modbus нет** — V3.3 не описывает запись целевой позиции через FC06. Для позиционирования через Modbus нужен отдельный документ или другой драйвер.
2. **FC03 адрес = P-xxx decimal** — подтверждено официальной документацией.
3. **FC41 EEPROM**: после отправки ждать минимум 5 секунд.
4. **Параметры сохраняются при FC06, но только в RAM** — при выключении питания без FC41 сбрасываются.

## Параметры коммуникации

| Параметр | Адрес | Значение | Описание |
|----------|-------|---------|----------|
| P-181 | 181 | 1 | Slave ID |
| P-182 | 182 | 2 | Baud rate (2 = 19200) |
| P-183 | 183 | 1 | Формат (1 = 8E1) |

> P-181/P-182/P-183 изменять только через панель на драйвере (FC06 запись принимается, но после смены baud связь может оборваться).
